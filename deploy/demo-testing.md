# Демо в интернете для тестов с клиентами (без дорогого облака)

Задача: открыть прототип с телефона, посмотреть поведение, собрать фидбек. **Vercel** при этом часто даёт высокий **TTFB** для пользователей из РФ: серверные ответы идут с удалённого региона (США / Западная Европа), плюс сеть до «холодного» serverless.

Ниже — практичные варианты проще и дешевле полноценного Yandex Cloud.

---

## Идея: приложение ближе к пользователю

Нужен **небольшой VPS** (виртуальный сервер) в датацентре с **хорошей задержкой до Москвы**:

| Вариант | Примерно | Комментарий |
|--------|----------|-------------|
| **Hetzner** (Хельсинки / Германия) | от ~4–5 €/мес | Часто выбирают под аудиторию РФ: низкая задержка, простая панель |
| **Timeweb Cloud / Selectel** (РФ) | от нескольких сотен ₽/мес | Оплата в рублях, поддержка на русском |
| **Railway / Render** (регион EU) | pay-as-you-go | Проще деплой, но не всегда дешевле VPS при постоянном демо |

База **может остаться на Supabase** (как сейчас): приложение на VPS подключается к ней по `DATABASE_URL`. Отдельно платить за «тяжёлое» облако под БД не обязательно.

## Минимальный путь: Docker на VPS

В репозитории уже есть **`Dockerfile`** и **`npm run docker:build`**.

1. Арендуйте VPS (1 vCPU, 1–2 GB RAM достаточно для прототипа).
2. Установите Docker на сервер (официальная инструкция для Ubuntu).
3. Скопируйте образ или соберите на сервере из git:
   - либо `docker build` прямо на VPS после `git clone`;
   - либо сборка у себя + `docker save` / частный registry / образ из CI — как удобнее.
4. Запуск (подставьте свои значения):

   ```bash
   docker run -d --name checkout -p 3000:3000 \
     -e DATABASE_URL="postgresql://..." \
     -e ADMIN_PASSWORD="..." \
     split-checkout:latest
   ```

5. **HTTPS и домен** (чтобы на телефоне не ругался браузер и был нормальный URL):
   - поставьте перед контейнером **Caddy** или **nginx + certbot**;
   - или используйте панель вроде **Coolify** / **Dokploy** на том же VPS — они часто сами поднимают TLS.

Для чистого демо можно временно открыть и **HTTP** по `http://IP:3000` — для внутренних показов иногда хватает, но для клиентов лучше HTTPS.

## Ещё проще на старте (без своего сервера)

- **Railway / Render / Fly.io** с выбором региона **ближе к EU-East** — иногда TTFB лучше, чем у дефолтного Vercel, но это не гарантия; смотрите замеры в DevTools с телефона.
- **Туннель с ноутбука** (`ngrok`, `cloudflared tunnel`) — годится на **одну встречу**, если не критична стабильность и скорость зависят от вашего интернета.

---

## Что померить перед выбором

В Chrome DevTools → **Network** → колонка **Waiting (TTFB)** на документе HTML и на `POST /api/checkout/scenario`. Цель демо — чтобы с телефона в 4G это было **терпимо** для UX-теста, не обязательно «как у продакшена».

## Итог

Для **недорогого постоянного демо** с адекватной задержкой для РФ чаще всего хватает **одного маленького VPS (EU-East или РФ) + ваш Docker-образ + Supabase как БД**. Yandex Cloud с управляемыми сервисами для этой задачи обычно избыточен.
