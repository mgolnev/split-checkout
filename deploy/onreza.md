# Деплой на ONREZA (https://app.onreza.ru/)

Проект: **Next.js 15** + **Prisma** + PostgreSQL. В репозитории есть **`Dockerfile`** (standalone, порт **3000**).

## 0. Автодеплой с GitHub

В настройках подключённого репозитория ONREZA должен быть **включён переключатель «Автоматический деплой»**. Если он выключен, пуш в `main` **не** запускает сборку — только ручной деплой из панели.

## 1. Переменные окружения в панели ONREZA

Добавьте как минимум:

| Ключ | Значение |
|------|-----------|
| `DATABASE_URL` | Строка `postgresql://...` (см. ниже про кодирование пароля) |
| `ADMIN_PASSWORD` | Свой пароль для `/admin` (не оставляйте `admin` в проде) |

`NODE_ENV=production` обычно выставляется платформой; дублировать не обязательно.

### Важно: пароль в `DATABASE_URL`

Символы **`@` `:` `%` `*` `#`** и др. в **пароле** нужно кодировать в URL (иначе подключение «тихо» ломается). Надёжный способ — взять пароль из панели БД и выполнить локально:

```bash
node -e "console.log('postgresql://USER:' + encodeURIComponent('ВАШ_ПАРОЛЬ') + '@HOST:5432/ИМЯ_БД?sslmode=require')"
```

`encodeURIComponent` **не** кодирует символ `*` (по правилам JS). Если в пароле есть `*`, замените в итоговой строке на `%2A` только в сегменте пароля (или закодируйте пароль вручную по таблице `%`→`%25`, `@`→`%40`, `*`→`%2A`).

Подставьте `USER`, `HOST`, `ИМЯ_БД` как в панели ONREZA / Kaiki. Скопируйте **целую** строку в переменную `DATABASE_URL`.

## 2. Сборка и запуск

- Если ONREZA собирает **Docker**: укажите репозиторий, ветку `main`, путь к **`Dockerfile`** в корне. Порт контейнера **3000** (в образе уже `EXPOSE 3000`, `PORT=3000`).
- Если сборка **без Docker** (Node): `npm ci` → `npm run build` → `npm start`. На этапе `build` к БД проект **не подключается**; `DATABASE_URL` нужен в **рантайме**.

## 3. Миграции Prisma (один раз после первого деплоя или после обновления схемы)

На машине, где есть доступ к этой же БД (ваш ПК с интернетом к хосту PostgreSQL):

```bash
cd /path/to/checkout
export DATABASE_URL='postgresql://...'   # та же строка, что в ONREZA
npm run db:migrate:deploy
```

Если ONREZA даёт **одноразовую консоль / SSH** с теми же переменными — можно выполнить там то же самое (нужны `node` + `npx prisma` или заранее собранный образ с CLI).

## 4. Данные в БД (опционально)

Пустая БД после миграций — чекаут и админка без товаров. Засеять демо-данные (с тем же `DATABASE_URL`):

```bash
export DATABASE_URL='postgresql://...'
npm run db:seed
```

Или используйте свой процесс наполнения через админку.

## 5. Проверка

- Откройте выданный ONREZA URL → `/checkout` и `/api/health` (если есть).
- Админка: `/admin/login` с паролем из `ADMIN_PASSWORD`.

### `/api/health` отдаёт `database: "down"`

В ответе смотрите **`prismaCode`** и **`hint`** (после деплоя актуальной версии приложения):

| prismaCode | Что обычно не так |
|------------|-------------------|
| **P1001** | С платформы ONREZA до хоста PostgreSQL **нет сети**: у провайдера БД (Kaiki и т.п.) нужно разрешить **внешние** подключения или добавить в allowlist **исходящие IP ONREZA** (уточните в поддержке ONREZA список IP / подсеть). |
| **P1000** | Неверные учётные данные или **пароль в `DATABASE_URL` не URL-encoded** (`@` → `%40`, `%` → `%25`, `*` → `%2A`). |
| *(нет кода)* | Переменная **`DATABASE_URL` не подхватилась** в рантайме (опечатка ключа, деплой без перезапуска, другой конфиг). |

Локально проверить строку подключения (с той же `DATABASE_URL`, что в панели):

```bash
export DATABASE_URL='postgresql://...'
npx prisma db execute --stdin <<< "SELECT 1"
```

## 6. Безопасность

После того как всё заработало, **смените пароль БД** в панели (кнопка «Сбросить пароль») и обновите `DATABASE_URL` в ONREZA с **новым** URL-encoded паролем. Пароль, который когда-либо светился в чате или скрине, считать скомпрометированным.
